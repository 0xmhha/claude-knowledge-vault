# claude-knowledge-vault — PLAN

> Local-first, privacy-preserving semantic + keyword search across the
> developer's own Claude Code conversation history. Sibling plugin to
> [claude-env-sync](../claude-env-sync). One-screen onboarding, MCP +
> dashboard + slash surfaces. Built on Claude Code's plugin lifecycle.

## 0. Vision (one line)

"Yesterday's hook decision? Type three words, get the exact turn, the
file, and the diff — in under a second, with nothing leaving the laptop."

## 2. Scope

### In scope (PoC v1)

- One-shot **indexer**: walk `~/.claude/projects/*.jsonl`, parse turns,
  chunk, persist into `${PLUGIN_DATA}/vault.db`.
- **Incremental** indexer: only re-read jsonl files whose `mtime > last_index_at`.
- **Search**: free-text query → ranked turn snippets with `(session_id,
  turn_index, source_path, timestamp, role)` pointer back to the jsonl.
- **MCP tools**: `kv_index`, `kv_search`, `kv_stats`, `kv_purge`.
- **Slash command**: `/kv <query>` — sugar over `kv_search`.
- **Dashboard**: search box + result list + turn-detail view. Localhost
  bind only (env-sync pattern).
- **Packaging**: `bin/run.sh` (env-sync shape) + cosign-signed releases.

### Out of scope (deferred to TODOS.md / v2)

- Realtime hook capture (UserPromptSubmit / Stop / PostToolUse). v1 reads
  jsonl that Claude Code already writes.
- RAG / "answer the question, don't just quote" (P1 explicitly = quote,
  not synthesise).
- Multi-machine sync of `vault.db` itself. If wanted, layer on env-sync —
  the db is just a file under `${CLAUDE_PLUGIN_DATA}`.
- Multi-user / team sharing.
- Embedded model bundling. v1 is BM25-only (see §5 retrieval); embedding
  add-on is v1.1 (a follow-up plan).
- Auto-deletion / retention policy. v1 indexes everything that exists.
- Cross-tool sources (gemini / copilot / aider history). v1 = Claude Code
  jsonl only.
- Export / import of the vault DB.

### Premise→scope traceability

- P1 (quote, not synthesise) → no RAG, just rank-and-display.
- P2 (single-user) → no auth, no multi-tenant schema, no shared writer locks.
- P3 (100 % local) → no upload, no external embed; offline-first DB.
- P4 (stack relax) → modernc.org/sqlite over CGO is preferred (no native
  toolchain) but a CGO build tag stays open for sqlite-vec in v1.1.
- P5 (jsonl read-only) → no settings.json mutation, no plugin hook
  install. Uninstall is `rm -rf ${CLAUDE_PLUGIN_DATA}`.
- P6 (3-surface) → MCP + dashboard + slash, same wiring as env-sync.

## 5. Key decisions

Numbered for reference from the Audit Trail.

### D1 — sqlite driver = `modernc.org/sqlite` (pure-Go transpile)

- **Why**: P3 (offline) + P4 (small vetted deps) + "no CGO toolchain on
  install" UX. `modernc.org/sqlite` ships FTS5 by default in recent
  versions. Same author maintains a GoVerify-able pure-Go transpile of
  upstream SQLite; binary is ~10 MB heavier but no native compiler
  required on the user's machine, which **matters for plugin install**.
- **Rejected**: `mattn/go-sqlite3` (CGO; toolchain required, slower install).
- **Rejected**: `crawshaw.io/sqlite` (CGO; smaller surface but same issue).
- **Rejected**: `sqlite3` binary subprocess (process-per-query latency).
- **Open**: a CGO build tag (`-tags=cgo_sqlite`) stays available for the
  v1.1 sqlite-vec path — same Go module, two builds.

### D2 — Retrieval v1 = BM25 only (no vector embed)

- **Why**: P3 forbids external embedding; running ONNX locally adds 80–
  300 MB of model weight + cold-start time. context-mode ships this
  exact problem (Claude conversation indexing) **with BM25 + Porter only**
  and it works. Start there; add embeddings only when a query class
  measurably fails.
- **Tokeniser**: `tokenize='porter unicode61'` (matches context-mode).
- **Boost**: title / role / source-label fields weight ≥ content; tune
  via FTS5 `bm25(content, title:5, role:2)` style aux.
- **Trigram side index**: `chunks_trigram` (`tokenize='trigram'`) — gives
  partial-word substring matches as a fallback for short queries.
- **Rejected v1**: sqlite-vec + local embed. Moved to v1.1 (see §13).
- **Rejected v1**: BM25 + reciprocal-rank-fusion ensemble. v1 returns
  BM25-only ranked list; trigram is a separate "did you mean" lane.

### D3 — Data source = jsonl read-only walk + mtime cache

- **Why**: P5. Uninstall is trivial. Reversible.
- **Trigger**: indexer runs on (a) `--once index`, (b) MCP `kv_index`
  tool, (c) dashboard "Re-index" button. **Not** auto-on-start to keep
  cold MCP boot fast.
- **Incremental**: `sources` table stores `(file_path, content_hash,
  last_mtime, chunk_count)`. Skip if hash matches.
- **Rejected**: hook-based realtime capture — moved to v2.

### D4 — Chunking = heading-aware + code-block-atomic, size-capped

- **Why**: Port of context-mode's algorithm (`src/store.ts:4–149`). Two
  rules: (1) split at `#` / `##` / `###` boundaries; (2) never split
  inside a fenced code block. Cap at ~2000 tokens or 8 KB whichever
  first — long chunks tank BM25.
- **Conversation-specific addition**: turn boundary is also a hard chunk
  boundary (one turn = one or more chunks; chunks never cross turns).
  Each chunk gets `(session_id, turn_index, role, ts)`.

### D5 — Surfaces: 3-surface, MCP-first, slash thin

- MCP tools (`kv_index`, `kv_search`, `kv_stats`, `kv_purge`). JSON-RPC
  over stdio (env-sync `internal/mcp` reused).
- Dashboard endpoints (mirror env-sync):
  - `GET  /api/stats`
  - `POST /api/index`
  - `GET  /api/search?q=...&limit=...&since=...&source=...`
  - `GET  /api/turn?session_id=...&turn_index=...`
  - `POST /api/purge`
  - `GET  /healthz`, `/`, `/style.css`, `/app.js` (embed.FS)
- Slash command: a thin Markdown command file that invokes the MCP tool;
  no extra runtime.

### D6 — Storage layout

```
${CLAUDE_PLUGIN_DATA}/
  vault.db                  # primary fts5 store
  vault.db-shm vault.db-wal # sqlite WAL files
  index.lock                # writer mutex (5 min stale)
  bin/<platform>/kvault     # binary cache (env-sync run.sh shape)
```

DB-file is the unit of backup / sync (P2 single-user, but if someone
*wanted* multi-machine: copy `vault.db` via env-sync, that's the only
moving part). Schema versioning via `PRAGMA user_version`.

### D7 — Security boundary

- DB and binary live under `${CLAUDE_PLUGIN_DATA}`, mode `0o700` dir + `0o600` files.
- Dashboard binds `127.0.0.1` only (env-sync rule).
- Search inputs go through FTS5 `MATCH` with `'"<phrase>"'` quoting; no
  raw concat. Path query params re-validated for traversal (env-sync's
  `exclude.ValidateNoTraversal` reused).
- Returned turn snippets are escaped via `textContent` in the web UI
  (env-sync XSS rule). Code-block content is rendered as `<pre>` but
  text-only.
- `vault.db` may contain quoted Claude turns including pasted secrets
  the user shared into the chat → **the DB itself is a sensitive file**.
  Document this in `SECURITY.md`. No silent upload anywhere (P3).
- **Out**: encryption-at-rest of the DB. PoC v1 trusts the user's
  filesystem (same as `~/.claude/` itself, which already holds
  `.credentials.json` etc.).

### D8 — Concurrency / lock

- One writer at a time. `index.lock` file (RFC3339 mtime, 5-min stale
  expiry) — exact copy of env-sync's `acquireLock`.
- SQLite WAL mode (`PRAGMA journal_mode=WAL`). Many concurrent readers,
  one writer.
- MCP `kv_search` and dashboard `GET /api/search` are read-only and
  hold no lock.

## 6. Failure modes & mitigations

| FM   | Category    | Symptom                                              | Mitigation                                                                                |
| ---- | ----------- | ---------------------------------------------------- | ----------------------------------------------------------------------------------------- |
| F-01 | Data        | Claude Code changes jsonl schema → parser breaks     | Schema fixture in `tests/fixtures/jsonl/*.jsonl`; CI snapshot. Parser is `internal/source`, single owner. |
| F-02 | Data        | jsonl turn order corrupted / interleaved             | `(session_id, turn_index)` PK enforces ordering; importer logs gaps as warnings, doesn't crash. |
| F-03 | Data        | One jsonl file 100 MB+                                | Stream-decode line-by-line (bufio.Scanner with 16 MiB max); chunk + write incrementally.  |
| F-04 | DB          | sqlite WAL grows unbounded                            | `PRAGMA wal_autocheckpoint = 1000`; nightly `wal_checkpoint(TRUNCATE)` via `kv_purge --wal`. |
| F-05 | DB          | Schema migration on new release                       | `PRAGMA user_version`; migration steps in `internal/store/migrations.go`; idempotent.     |
| F-06 | Privacy     | DB leaked to upload / cloud-sync                      | Document in SECURITY.md. Ship `tests/regression-no-network.sh` — runs binary under `nettest` denying all egress, asserts indexing + search still work. |
| F-07 | Privacy     | User's pasted secret indexed and shown in turn quote  | Search results re-mark obvious patterns (`sk-*`, `AKIA*`, JWT shape) before render — same regex set as env-sync `internal/exclude/patterns.go`. |
| F-08 | UX          | Empty result, BM25 score weak                         | Trigram fallback lane shows "Did you mean?" snippets. Result UI shows BM25 score histogram. |
| F-09 | UX          | Slow first index (~1000 sessions)                     | Indexer reports progress via SSE (`event: progress`). Resumable via `last_mtime` on each file. |
| F-10 | Build       | modernc.org/sqlite Go-transpile drops FTS5 build flag | Pin a known-good version; CI smoke `kv_search "hello"` against an empty DB; assert no `no such module: fts5` error. |
| F-11 | Build       | Plugin install on a fresh machine without Go         | `bin/run.sh` source-build path still requires Go (same as env-sync); release-fetch path solves this once T-P.4 ships. |
| F-12 | Concurrency | Two `kv_index` MCP calls collide                      | Lock file (env-sync pattern). Second caller gets `ErrIndexInProgress`, no corruption.    |
| F-13 | Adversarial | jsonl file is symlink to `/etc/passwd`                | Walk uses `os.Lstat`; skip non-regular files; log.                                       |
| F-14 | Adversarial | Crafted jsonl with 1 MB single line                   | Scanner buffer 16 MiB cap; oversize line is logged + skipped, not crashed.               |

## 8. Test strategy

### Unit (Go testing stdlib)

- `source`: jsonl fixture decode (5–10 captured turn shapes) + corrupt-
  line skip + EOF.
- `chunk`: heading split, code-block-atomic, size cap, turn-boundary
  guard.
- `store`: schema creation, idempotent migration, FTS5 MATCH against
  golden corpus.
- `search`: BM25 ranking stable on sorted-by-score golden test;
  trigram fallback fires on short queries.
- `indexer`: full pipeline against fixtures, asserting (chunks_count,
  sessions_count, last_mtime advance).

### Integration (Go `_test.go`, real sqlite)

- index → search → assert top-3 contains seeded query.
- Re-index unchanged jsonl → 0 rows touched (mtime gating).
- Re-index after appending one turn → 1 new chunk only.
- Concurrent reader during write does not block (WAL mode).

### Regression (bash)

- `regression-no-network.sh` — under `nettest` / unshare-net, run
  index + search, both must pass. Proves P3.
- `regression-secret-rerender.sh` — index a turn containing
  `sk-proj-CANARY-DEAD-BEEF`, run search, assert the rendered
  dashboard response masks the pattern.

### Web smoke (manual + CI shellcheck)

- ASCII screenshot in README (env-sync precedent).
- CI starts `kvault --port 0`, curls `/`, `/api/stats`, `/api/search`,
  asserts 200 + JSON shape.

### Cover target

- 80 % statements per package (matches env-sync).
- `cmd/kvault`: 30 %+ acceptable (thin glue).

  it back to user with citation.

### TTHW (Time to Hello World)

Target: **under 90 seconds** from `git clone` to first successful search.

```bash
git clone …/claude-knowledge-vault.git
cd claude-knowledge-vault
( cd go && go build -o ../kvault ./cmd/kvault )
./kvault --once index --plugin-data /tmp/kv
./kvault --once search --plugin-data /tmp/kv --query "the thing"
```

Five commands, zero config files, zero external accounts.

### Error message rule

Every error follows env-sync's *problem + cause + fix* format. Examples:

- `index failed: no jsonl files found at ~/.claude/projects/ (Claude
  Code has not yet written any session). Start a Claude Code session,
  then re-run.`
- `search failed: query is empty (--query was blank or stdin closed).
  Provide a query string.`

### API/CLI naming consistency

- All MCP tools prefixed `kv_` (no namespace collision with env-sync's
  `sync_*`).
- CLI flags mirror env-sync: `--once`, `--mcp`, `--port`,
  `--plugin-data` (analogous to `--claude-dir`).
- Exit codes inherit env-sync's: 0 ok, 1 user, 2 runtime, 3 conflict
  (unused here but reserved), 4 build.

## 14. Open items for human gate (Phase 4)

After 6-principle auto-decide, these survive as **taste decisions** the
user can override. No User Challenges — every premise was user-stated
and reviewers agree.

### T1 — `modernc.org/sqlite` vs CGO sqlite3 default

- Auto-decision: **modernc.org/sqlite** (D1 rationale).
- Tradeoff: pure-Go = no native toolchain on install (huge UX win) but
  ~30 % slower than CGO sqlite3 on bulk insert. Bulk insert happens
  once per re-index; search latency is the same.
- Override path: switch to `mattn/go-sqlite3` if first-index of 10 k
  sessions takes longer than ~3 min on M-class laptop.

### T2 — One binary (`kvault`) or two (`kvault-indexer`, `kvault-server`)

- Auto-decision: **one binary**, env-sync precedent.
- Tradeoff: one binary is simpler to ship and matches user mental
  model. Two binaries would let people index on a server-class box and
  search locally — but P2 (single-user, single-machine) makes that a
  non-feature.
- Override: split if remote-index becomes a use case.

### T3 — Snippet length policy

- Auto-decision: **return 240 char windows** centred on the BM25 match,
  with `…` ellipsis on either side. Result list keeps low.
- Tradeoff: shorter snippets may miss critical context; longer
  snippets blow up the result page.
- Override: configurable via `--snippet-chars`.

### T4 — Slash command granularity

- Auto-decision: **single `/kv <query>`** (thin sugar over `kv_search`).
- Tradeoff: more commands (`/kv-recent`, `/kv-source`, `/kv-session`)
  surface more capability but clutter the slash menu. User can pass
  the same filters as MCP args.
- Override: add more slash commands if telemetry shows they get used.

### T5 — Indexing trigger UX

- Auto-decision: **manual trigger** (CLI / MCP `kv_index` / dashboard
  "Re-index" button). No background daemon.
- Tradeoff: requires the user to remember to re-index. A background
  watcher (fsnotify) would auto-pickup new jsonl files but means a
  long-running process the user didn't ask for.
- Override: add a `--watch` mode in v1.1 if users ask.

### T6 — Re-mark secrets in rendered turns?

- Auto-decision: **yes, mark with the env-sync regex set** (sk-*, AKIA*,
  JWT shape). Render the masked variant; "show original" button toggles.
- Tradeoff: false positives mask legitimate strings; false negatives
  leak. Mask-then-toggle is the safer side.
- Override: disable via `--no-mask` for users who store no secrets in
  Claude chats.
